# Projeto Prático de QA – Testes Manuais no Habit Tracker

Realizei um projeto prático de testes manuais na aplicação **Habit Tracker**, com o objetivo de identificar defeitos e validar o funcionamento das principais funcionalidades do sistema.

O foco do projeto foi garantir que os fluxos de **login**, **adição de tarefas**, **edição de tarefas** e **exclusão de tarefas** estivessem funcionando corretamente, além de verificar a experiência do usuário e a consistência da interface.

Durante a execução, apliquei diferentes técnicas e tipos de testes, incluindo:

* Testes funcionais para validar os requisitos e o comportamento esperado das funcionalidades.
* Testes de Interface (UI) para verificar layout, alinhamento, responsividade e consistência visual.
* Testes de validação de campos para conferir regras de entrada, obrigatoriedade, limites e mensagens de erro.
* Testes negativos utilizando entradas inválidas e cenários inesperados para avaliar o tratamento de erros pela aplicação.
* Testes exploratórios para identificar comportamentos não previstos e possíveis falhas fora dos cenários planejados.
* Testes de regressão (quando aplicável) para confirmar que alterações realizadas não afetaram funcionalidades já existentes.

## Ferramentas utilizadas

Como apoio à execução dos testes, utilizei o **Chrome DevTools** para inspecionar elementos da interface, analisar requisições e respostas na aba Network, verificar mensagens de erro no Console e auxiliar na investigação de possíveis defeitos.

Ao longo do projeto, documentei casos de teste, registrei evidências dos testes executados e elaborei relatórios de bugs contendo descrição do problema, passos para reprodução, resultado esperado, resultado obtido e prioridade da correção.

**Objetivo do projeto:** Validar o correto funcionamento das funcionalidades de login, adicionar tarefas, editar tarefas e apagar tarefas, identificando defeitos e contribuindo para a melhoria da qualidade da aplicação.
